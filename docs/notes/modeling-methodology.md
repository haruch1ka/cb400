# ハードサーフェスモデリングの方法論

作成日: 2026-06-18

---

## ハードサーフェスモデリングとは何か

バイク・車・ロボット・建築物——こうした「硬い人工物」を3Dで作ることをハードサーフェスモデリングと呼ぶ。有機物（人体・植物）のスカルプトとは異なり、機械的な精度と直線・曲面の共存が求められる。

CB400SF のようなバイクは、その典型だ。フレームパイプの円筒、タンクのなめらかな曲面、エンジンブロックの直線的な塊、細かいボルト類——一台の中に全種類の難しさが詰まっている。

---

## 2つのワークフロー

ハードサーフェスモデリングには大きく2つのアプローチがあり、パーツの性質によって使い分ける。

### Subdivision Surface（サブサーフ）ワークフロー

少ないポリゴンで粗いメッシュを作り、Subdivision Surface モディファイアをかけて滑らかにする方法。ポイントは「**モディファイアをかける前の形**で最終形状を制御する」こと。

滑らかすぎる箇所には **サポートループ**（エッジの近くに追加するループ）を入れて形を引き締める。エッジから近いほどシャープに、遠いほどなだらかになる。あるいは `Shift+E` の **Edge Crease** で追加ジオメトリなしにシャープさを与えることもできる。

このワークフローの大前提は「**Quad（四角ポリゴン）しか使わない**」こと。三角形やN-gon（五角形以上）が混じると、Subdivide をかけたときに予測不能な膨らみや凹みが生じる。トポロジーを綺麗に保つことがそのまま品質に直結する。

CB400SF での適用例：タンク、シートカウル、フェンダー、ライト類、フレームパイプ、ホイール。

### Boolean（ブーリアン）ワークフロー

カット用のオブジェクトを別途作り、Boolean モディファイア（Difference / Union）でベースメッシュを切り抜く方法。パネルライン・ネジ穴・通気口など、「形状に穴や溝を開ける」作業が圧倒的に速くなる。

モディファイアとして積んでおく限り**非破壊的**に作業できるのが強みだ。ただしカット後のトポロジーは崩れやすく、そのまま Subdivision をかけると破綻する。最終的にクリーンアップするか、あえて Subdivision を使わない方針にするかを決めておく必要がある。

CB400SF での適用例：エンジンブロックの細部、メーターパネル、ボルト穴のカット。

### 両者の組み合わせ

実際のワークフローでは2つを組み合わせる。「大まかな曲面は SubSurf で作り、細部のパネルラインは Boolean で切る」というのが典型的な進め方だ。エンジンブロックなどはその好例で、全体の塊は SubSurf、冷却フィンの溝は Boolean で処理するのが自然になる。

## ブロッキングという考え方

Phase 1（ブロッキング）は、プロポーションと全体シルエットだけを固める工程だ。細部は一切作らない。

なぜこれが重要かというと、**ブロッキングの狂いは後工程で修正できない**からだ。タンクの横幅が 5mm 広すぎたまま細部まで作り込んでしまうと、修正コストは何倍にも膨れる。シルエットが「実車に見える」状態になってから初めて次へ進む。

作業の順番には理由がある。まずフレームを作るのは、フレームが全パーツの位置基準になるからだ。次にホイールを作ることで接地面と車高が決まり、フレームとの整合性を確認できる。エンジンはその後、フレーム内に収まるかを見ながら作る。

各ブロッキングはシンプルなプリミティブ（Cube / Cylinder / Sphere）の変形で十分だ。リファレンス画像に重ねて正面・側面の両方で形が合っているか確認し、カメラビューで実車写真と見比べる。「これでいい」と思えてから次へ。

## トポロジーを意識する

トポロジーとは頂点・エッジ・面の接続関係のことだ。綺麗なトポロジーは Subdivide 後の品質に直接影響し、後工程の UV 展開も楽になる。

**Quad を使う理由**は3つある。Subdivision が均等にかかること、エッジフローが読みやすいこと、予測通りのシェーディングになること。逆に N-gon は Subdivide 時に内部で三角形に変換され、意図しないポールを生む。ポールとは5本以上のエッジが一点に集中する頂点のことで、SubSurf をかけるとそこだけ引きつれる（ピンチング）。

ポールは避けられない場合もあるが、**平坦な面に置く**のが鉄則だ。曲面の頂点付近にポールがあるとシェーディングの崩れが目立つ。

## モディファイアの運用方針

CB400SF でよく使うモディファイアと、その使い方の勘所を押さえておく。

**Mirror** は左右対称パーツを効率よく作るための基本。片側だけ作ればよいので作業量が半分になる。ただし適用は最終段階まで保留する。適用してしまうと左右を独立して編集できなくなる。

**Subdivision Surface** はビューポートのレベルを 2 に保ち、レンダーは 3 にするのが目安。作業中からレンダーレベルにすると重くなるため、確認時だけ上げる。

**Bevel** はエッジの面取りに使う。Width を小さく（0.5mm 程度）、Segments を 2〜3 にすることで実物らしいエッジハイライトが生まれる。鋭すぎるエッジは CG っぽく見える。

**Array** はスポーク・ボルト・冷却フィンなど繰り返しパーツに使う。Count か Fit Curve で数・間隔を制御する。

**Screw** はフレームパイプのような円筒断面を持つ形状に向く。断面プロファイル（2D カーブ）を先に作ってから適用する。

**Shrinkwrap** はケーブル・ハーネスを別メッシュの表面に沿わせるときに使う。Phase 3（ディテール）で活躍する。

## モデリング方針

### 基本方針
1. **参照点を先に置く** — 実寸法でエンプティ（Empty）を配置し、スケールの基準にする
2. **対称活用** — 左右対称パーツはミラーモディファイアを使い、片側だけ作る
3. **ブロッキング優先** — 細部より全体プロポーションを先に固める
4. **コレクション管理** — パーツをコレクションで整理（Engine / Frame / Body / Wheels / Electrical）
5. **モディファイア活用** — Subdivision Surface, Mirror, Bevel, Screw を積極使用

### パーツ分割戦略

```
CB400SF
├── Frame（フレーム）
│   ├── main_frame        ← ダイヤモンドフレーム本体
│   ├── subframe          ← シート下サブフレーム
│   └── engine_mount      ← エンジンマウント
├── Engine（エンジン）
│   ├── cylinder_head     ← シリンダーヘッド
│   ├── cylinder_block    ← シリンダーブロック
│   ├── crankcase         ← クランクケース
│   ├── intake            ← 吸気系
│   └── exhaust           ← 排気管
├── Body（外装）
│   ├── fuel_tank
│   ├── seat_cowl
│   ├── side_cover_L/R
│   ├── fender_front
│   └── fender_rear
├── Suspension（足回り）
│   ├── front_fork_L/R
│   ├── top_bridge
│   ├── swingarm
│   └── rear_shock
├── Wheels（ホイール）
│   ├── wheel_front       ← リム + スポーク + ハブ
│   ├── tire_front
│   ├── wheel_rear
│   └── tire_rear
└── Electrical（電装）
    ├── headlight
    ├── taillight
    ├── meter_panel
    └── turn_signal_x4
```

## 作業順序（推奨）

1. **スケール確認用エンプティ配置**（全長2080mm基準）
2. **リファレンス画像設定**（正面・側面）
3. **フレーム** → プロポーションの骨格になる
4. **ホイール** → 地面との接点、高さの基準
5. **エンジン** → 最も複雑、早めに着手
6. **外装** → フレーム・エンジンに合わせて形を出す
7. **ディテール** → ボルト・ケーブル等は最後

## よくある落とし穴と対策

| 落とし穴 | 対策 |
|----------|------|
| スケールがバラバラになる | Ctrl+A でスケール適用を徹底 |
| ポリゴンが増えすぎて重くなる | コレクションの表示/非表示を活用 |
| 後からのUV展開が大変 | 各パーツ完成時に即UV展開 |
| エンジン細部で行き詰まる | 外から見える面だけモデリング（内部は省略） |
| 左右非対称ミスが起きる | ミラーモディファイア適用は最後まで保留 |
